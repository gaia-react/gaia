This project uses the GAIA preset. Bare `/speckit-clarify` is not the clarify path here: core clarify writes an off-shape artifact (a `## Clarifications` / `### Session` block with five-word answers) and carries a question cap GAIA does not use. Run `/gaia-spec` instead — it drives GAIA's coverage-based Socratic clarify loop (one question per turn) and folds answers into the SPEC frontmatter `clarifications.answered[]` in GAIA's shape. See `.specify/extensions/gaia/templates/clarify-prompts.md` and `.claude/skills/gaia/references/spec.md`. Do not run the core clarify loop; hand off to `/gaia-spec`.
